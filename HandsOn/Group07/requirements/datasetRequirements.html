Dataset requirements

Requirements satisfied

  • R1. Data are in one of the selected smart city domains.

    The dataset shows the quality of air in Madrid, which is one of the available domains to be covered

  • R2. Data are available as a CSV file.

    Every file in the dataset has CSV format.

  • R3. Data have an open license so they can be published.

    The dataset has an open license, has written below

  • R4. Data can be easily linked with generic real-world entities (e.g. locations).

    The dataset has many entities that can be linked, as written below.

Source and license

The "air quality" data present in this dataset comes from "Madrid's City Council Open Data" website (link).
The data therefore inherits the "Madrid Open Data Terms of Use" (link) to the spanish detailed version of the license, which allows for free commercial and non-commercial use, and provides no liability on the data.
For example, permitted activities include copying, dissemination, modification, adaptation, extraction, reordering and combination of information belonging to the dataset.

Entities that can be linked with other datasets

  • Address

    The address of the pollution measurement stations can be linked to a dataset of locations in Madrid.

  • Pollutant name

    The pollutant name can be linked to a description of the pollutant.

Other dataset that can be linked to the entities

We can use information that can be correlated to the position (address) of the measurements

  • Buildings belonging to companies or Public Administration that can produce pollution with their industrial processes
  • Traffic statistics about the same area
